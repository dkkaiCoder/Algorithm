# 构建体系化算法

## 📖 背景 (Situation)

在计算机科学的学习中，算法题往往止步于“AC”。我发现普遍存在三个问题：
1.  **遗忘快**：刷过即忘，重复劳动。
2.  **无体系**：知识点零散，无法串联。
3.  **难复用**：解题技巧无法沉淀为可迁移的思维模型。

我需要一个 **能对抗遗忘、支撑决策、沉淀方法论** 的算法学习系统。

## 🎯 任务 (Task)

建立一个 **“体系化算法笔记库”**，核心标准如下：
- **可追溯**：每一份记录都能清晰追溯解题思路与决策逻辑。
- **可复用**：抽象出通用的代码模板与解题模式。
- **可演化**：能够不断重构、优化、加入新感悟。
- **抗遗忘**：通过周期性回顾与 Markdown 标注，固化长期记忆。
